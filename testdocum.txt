# СТРАНИЦА 1

# ВВЕДЕНИЕ

В современном мире многопользовательские компьютерные игры являются не только способом развлечения, но и полноценной средой социального взаимодействия между пользователями. Развитие игровой индустрии привело к увеличению числа командных онлайн-игр, в которых успешность игрового процесса напрямую зависит от качества взаимодействия между участниками команды. Однако одной из наиболее актуальных проблем современных игроков остаётся поиск подходящих тимейтов для совместной игры. Многие игровые платформы не предоставляют удобного механизма подбора игроков по интересам, игровому опыту, возрасту или времени активности. В результате пользователи вынуждены искать напарников через сторонние мессенджеры, игровые форумы, социальные сети или случайный подбор внутри самой игры, что значительно усложняет процесс организации совместного игрового процесса.

В связи с этим была поставлена задача разработки мобильного приложения **TeamUp**, предназначенного для поиска тимейтов в многопользовательских играх. Основной целью проекта является создание удобного мобильного сервиса, позволяющего пользователям быстро находить игроков со схожими интересами, совпадающими игровыми предпочтениями и комфортными условиями взаимодействия. Разрабатываемое приложение ориентировано исключительно на мобильную платформу Android и предоставляет пользователю возможность создания собственного профиля, указания игровых интересов, количества игровых часов, возраста и города проживания, а также поиска потенциальных партнёров для совместной игры.

Приложение TeamUp реализует современную механику взаимодействия между пользователями, основанную на системе лайков. Пользователь имеет возможность просматривать анкеты других игроков и ставить лайки заинтересовавшим профилям. В случае взаимной симпатии формируется совпадение (match), после чего пользователям становится доступен встроенный чат, предназначенный для дальнейшего общения и координации совместной игры. Подобная механика позволяет исключить нежелательное взаимодействие между пользователями и повысить качество коммуникации внутри платформы.

Для реализации серверной части приложения использовался фреймворк Django, обеспечивающий высокую скорость разработки, безопасность и удобство сопровождения проекта. Для организации взаимодействия между клиентской и серверной частью был применён Django REST Framework, реализующий REST API и передачу данных в формате JSON. В качестве системы хранения данных была выбрана SQLite, которая обеспечивает достаточную производительность для учебного проекта и позволяет упростить развёртывание серверной инфраструктуры.

Клиентская часть мобильного приложения была разработана с использованием React Native и Expo, что позволило ускорить процесс разработки мобильного интерфейса и обеспечить стабильную работу приложения на Android-устройствах. Для обмена данными между клиентской и серверной частью использовалась библиотека Axios, обеспечивающая выполнение HTTP-запросов и получение информации с сервера в режиме реального времени.

Развёртывание серверной части осуществлялось на VPS-хостинге Beget. Для обеспечения стабильной работы веб-приложения использовался сервер приложений Gunicorn и веб-сервер Nginx, выполняющий функцию обратного прокси-сервера. Подобная архитектура обеспечивает корректную обработку клиентских запросов и позволяет гарантировать стабильность функционирования мобильного приложения.

Таким образом, разработка приложения TeamUp является актуальной задачей, направленной на решение проблемы поиска тимейтов для онлайн-игр посредством создания удобной платформы для поиска, общения и организации совместного игрового процесса.

# 1. ТЕХНИЧЕСКОЕ ЗАДАНИЕ

## 1.1 Назначение разработки

Программное средство TeamUp предназначено для поиска тимейтов в многопользовательских компьютерных играх. Основной целью разработки является создание мобильного приложения, предоставляющего пользователям возможность быстро находить партнёров для совместной игры на основе совпадения интересов и игровых предпочтений. Система ориентирована на игроков, испытывающих трудности при поиске постоянной команды или желающих расширить круг игровых знакомств.

Приложение обеспечивает удобный механизм взаимодействия между пользователями посредством системы лайков и взаимных совпадений. После установления взаимного интереса пользователям становится доступен встроенный чат, позволяющий организовать совместный игровой процесс без необходимости перехода на сторонние платформы или мессенджеры.

Основными задачами приложения являются создание пользовательских профилей, хранение информации об игровых предпочтениях, реализация механизма поиска игроков, обеспечение безопасного взаимодействия между пользователями, а также организация встроенной системы общения.

## 1.2 Основание для разработки

Основанием для разработки программного продукта TeamUp послужила необходимость создания специализированной платформы, предназначенной для поиска игровых партнёров. Анализ существующих решений показал, что большинство игровых сервисов либо не имеют встроенной системы поиска тимейтов, либо реализуют её недостаточно эффективно. Поиск игроков часто осуществляется через сторонние ресурсы, что создаёт дополнительные сложности для пользователей и увеличивает время организации игрового процесса.

В связи с этим было принято решение о создании отдельного мобильного приложения, объединяющего функции поиска игроков, формирования игровых знакомств и встроенной коммуникации.

## 1.3 Функциональные требования

Разрабатываемая система должна обеспечивать выполнение следующих функций:

— регистрация нового пользователя посредством ввода логина и пароля;

— авторизация зарегистрированного пользователя;

— создание и редактирование пользовательского профиля;

— указание игровых часов;

— добавление информации о предпочитаемых играх;

— указание возраста пользователя;

— указание города проживания;

— просмотр анкет других пользователей;

— возможность установки лайков;

— определение взаимного совпадения между пользователями;

— автоматическое открытие встроенного чата после взаимного лайка;

— обмен сообщениями между пользователями;

— хранение пользовательских данных в базе данных.

Функциональность системы должна быть реализована таким образом, чтобы обеспечить максимально простой и интуитивно понятный интерфейс для конечного пользователя.

## 1.4 Требования к надёжности

Разрабатываемая система должна обеспечивать стабильное функционирование как клиентской, так и серверной части приложения. При возникновении ошибок соединения пользователь должен получать уведомление о невозможности подключения к серверу. В случае неверного ввода логина или пароля система должна информировать пользователя об ошибке авторизации.

Серверная часть должна корректно обрабатывать одновременно поступающие HTTP-запросы, исключая потерю пользовательских данных. Особое внимание уделяется сохранности информации о профилях пользователей, лайках, совпадениях и переписках.

При разработке приложения использовались встроенные механизмы безопасности Django, позволяющие снизить вероятность возникновения ошибок безопасности и обеспечить защиту серверной части от несанкционированного доступа. Передача информации между клиентом и сервером осуществляется через REST API, что позволяет стандартизировать обмен данными и повысить надёжность взаимодействия компонентов системы.

## 1.5 Требования к программным средствам

Для функционирования серверной части системы требуется следующее программное обеспечение:

— операционная система Linux;

— Python версии 3.10 и выше;

— Django;

— Django REST Framework;

— SQLite;

— библиотека corsheaders;

— Gunicorn;

— Nginx.

Для клиентской части требуется:

— Android-устройство;

— установленное приложение TeamUp APK;

— доступ к сети Интернет.

Разработка клиентской части осуществлялась с использованием React Native и среды Expo, что позволило обеспечить кроссплатформенный подход разработки мобильных интерфейсов и значительно ускорить процесс программирования.

## 1.6 Требования к аппаратным средствам

Для корректного функционирования клиентской части достаточно мобильного устройства на базе операционной системы Android с доступом к сети Интернет.

Серверная часть размещается на виртуальном частном сервере VPS, предоставляемом хостинг-провайдером Beget. Использование VPS позволяет обеспечить круглосуточную доступность приложения, стабильную обработку пользовательских запросов и централизованное хранение данных.

# СТРАНИЦА 4

# 2. ОПИСАНИЕ ПРОГРАММЫ

Приложение TeamUp представляет собой мобильную систему поиска тимейтов, функционирующую по принципу подбора пользователей на основании совпадающих игровых интересов. Главной особенностью приложения является использование механизма взаимного лайка, позволяющего открывать чат исключительно между пользователями, проявившими взаимную заинтересованность.

После запуска приложения пользователь проходит процедуру регистрации. Для создания учётной записи необходимо указать логин и пароль. После успешной регистрации система предоставляет возможность заполнения пользовательского профиля.

В профиль пользователя входят следующие параметры:

— игровые часы;

— предпочитаемые игры;

— возраст;

— город проживания.

Наличие подобной информации позволяет другим пользователям быстрее оценить совместимость потенциального тимейта и принять решение о дальнейшем взаимодействии.

После заполнения анкеты пользователю становится доступен раздел поиска игроков. Система отображает доступные профили, которые можно просматривать и оценивать. Если пользователь заинтересован в совместной игре, он может поставить лайк выбранному профилю.

Система лайков является односторонней. Это означает, что пользователь может поставить лайк любому человеку, однако возможность общения открывается только в случае взаимного интереса. Когда оба пользователя поставили лайк друг другу, автоматически формируется совпадение (match), после чего появляется доступ ко встроенному чату.

Подобная логика взаимодействия позволяет минимизировать количество нежелательных сообщений и делает процесс знакомства более комфортным для обеих сторон.

# СТРАНИЦА 5

# 3. АРХИТЕКТУРА ПРОГРАММНОГО ПРОДУКТА

Архитектура приложения TeamUp построена по принципу клиент-серверного взаимодействия. Данный подход предполагает разделение программного продукта на две независимые части: серверную и клиентскую.

Серверная часть отвечает за хранение информации, обработку пользовательских запросов, авторизацию, управление базой данных, обработку лайков, формирование совпадений и обеспечение работы системы сообщений.

Клиентская часть представляет собой мобильное приложение, устанавливаемое на Android-устройство. Пользователь взаимодействует исключительно с мобильным интерфейсом, а все вычислительные процессы, связанные с обработкой информации, выполняются сервером.

Взаимодействие между клиентской и серверной частью осуществляется посредством REST API. При выполнении любого действия, например авторизации, поиска пользователей или отправки сообщения, клиентское приложение отправляет HTTP-запрос на сервер. Сервер обрабатывает поступившую информацию, взаимодействует с базой данных и возвращает результат в формате JSON.

Использование REST API позволяет разделить ответственность между компонентами системы и значительно упрощает поддержку программного продукта. Благодаря подобной архитектуре приложение может масштабироваться и модернизироваться без необходимости полной переработки системы.

В качестве серверной технологии использовался Django — один из наиболее популярных Python-фреймворков для веб-разработки. Выбор Django обусловлен его высокой безопасностью, встроенной системой маршрутизации, наличием ORM и возможностью быстрого создания серверной логики.

## 3.1 Серверная часть приложения

Серверная часть программного продукта TeamUp реализована на основе веб-фреймворка Django. Выбор данной технологии обусловлен рядом преимуществ, среди которых высокая скорость разработки, встроенные механизмы безопасности, удобная работа с базами данных и возможность быстрого масштабирования проекта. Django предоставляет разработчику готовую архитектуру, позволяющую сосредоточиться непосредственно на реализации бизнес-логики приложения.

Для организации обмена данными между мобильным приложением и сервером использовался Django REST Framework (DRF). Данный инструмент представляет собой расширение Django, предназначенное для построения REST API. Использование DRF позволило реализовать передачу данных между сервером и мобильным приложением в формате JSON, что существенно упростило взаимодействие между компонентами системы.

Основной задачей серверной части является обработка клиентских запросов. Когда пользователь выполняет определённое действие в мобильном приложении, например авторизуется, просматривает анкеты, ставит лайк или отправляет сообщение, приложение формирует HTTP-запрос и передаёт его серверу. Сервер принимает запрос, обрабатывает полученные данные, взаимодействует с базой данных и отправляет клиенту соответствующий ответ.

Серверная часть также отвечает за проверку корректности пользовательских данных. Например, при регистрации система проверяет уникальность логина, корректность заполнения обязательных полей и соответствие формата передаваемой информации установленным требованиям. Подобный подход позволяет минимизировать вероятность возникновения ошибок и обеспечивает стабильность работы приложения.

Дополнительно сервер отвечает за реализацию логики совпадений. После постановки лайка система выполняет проверку наличия встречного лайка от другого пользователя. В случае обнаружения взаимного интереса автоматически создаётся совпадение (match), а также открывается доступ к системе сообщений.

Для обеспечения взаимодействия между мобильным приложением и сервером использовалась библиотека corsheaders. Данный инструмент позволяет корректно обрабатывать междоменные запросы, что особенно важно при клиент-серверной архитектуре, где мобильное приложение взаимодействует с отдельным сервером.

Таким образом, серверная часть TeamUp обеспечивает безопасную обработку пользовательских запросов, управление пользовательскими данными и реализацию ключевой бизнес-логики приложения.

## 3.2 База данных и организация хранения информации

В качестве системы управления базой данных в проекте была выбрана SQLite. Данное решение обусловлено относительной простотой настройки, высокой скоростью работы и отсутствием необходимости установки дополнительного серверного программного обеспечения. SQLite представляет собой встроенную реляционную базу данных, функционирующую в виде отдельного файла внутри проекта.

Использование SQLite является рациональным решением для учебного проекта, поскольку позволяет упростить процесс разработки и развертывания серверной части. При необходимости дальнейшего масштабирования приложения архитектура проекта позволяет перейти на более производительные системы управления базами данных, например PostgreSQL, без существенного изменения логики приложения.

База данных TeamUp предназначена для хранения всей информации, необходимой для функционирования программного продукта. В частности, система содержит сведения о зарегистрированных пользователях, параметрах профиля, лайках, совпадениях и сообщениях.

Каждый пользователь обладает индивидуальным профилем, содержащим логин, пароль, список предпочитаемых игр, количество игровых часов, возраст и город проживания. Данные параметры используются системой для отображения анкет и последующего поиска тимейтов.

Дополнительно база данных хранит информацию о поставленных лайках. Каждый лайк фиксируется отдельно, что позволяет реализовать механизм взаимных совпадений. После обнаружения встречного лайка система автоматически формирует match и предоставляет доступ ко встроенному чату.

Отдельная сущность базы данных предназначена для хранения сообщений пользователей. Каждое сообщение содержит информацию об отправителе, получателе, времени отправки и текстовом содержимом. Это обеспечивает возможность полноценного общения между пользователями после взаимного совпадения.

Благодаря использованию ORM, встроенной в Django, взаимодействие с базой данных значительно упрощается. Разработчику не требуется вручную писать SQL-запросы, поскольку все операции выполняются через модели Django, что повышает читаемость программного кода и снижает вероятность возникновения ошибок.

Использование структурированной базы данных обеспечивает централизованное хранение информации и гарантирует корректную работу всех основных функций приложения.

## 3.3 Клиентская часть приложения

Дополнительно в процессе разработки использовалась платформа Expo, существенно упрощающая запуск, тестирование и отладку мобильного приложения. Expo предоставляет удобный набор инструментов для работы с React Native и позволяет значительно ускорить процесс разработки.

Мобильное приложение TeamUp ориентировано на устройства под управлением операционной системы Android. После установки APK-файла пользователь получает доступ к интерфейсу приложения и может пройти регистрацию.

Интерфейс приложения был спроектирован таким образом, чтобы обеспечить максимальную простоту взаимодействия. После входа в приложение пользователь получает доступ к личному профилю, где отображается основная информация, необходимая для поиска тимейтов. Пользователь может указать предпочитаемые игры, количество игровых часов, возраст и город проживания. После возникновения взаимного совпадения пользователю становится доступен встроенный чат. Это позволяет оперативно обсудить детали совместной игры, договориться о времени подключения и установить контакт без использования сторонних сервисов.

Для взаимодействия с серверной частью использовалась библиотека Axios. Она отвечает за отправку HTTP-запросов и получение данных с сервера. Axios обеспечивает удобную обработку ответов и значительно упрощает реализацию клиент-серверного взаимодействия.

## 3.4 Алгоритм работы приложения

Работа мобильного приложения TeamUp начинается с регистрации пользователя. Для создания учётной записи необходимо указать логин и пароль. После успешного завершения регистрации пользователь получает возможность входа в систему.

Следующим этапом является заполнение пользовательского профиля. Пользователь указывает предпочитаемые игры, количество игровых часов, возраст и город проживания. Данная информация используется для повышения эффективности поиска подходящих тимейтов.

После заполнения анкеты становится доступен раздел поиска игроков. Система отображает анкеты других пользователей, содержащие основную информацию о потенциальных партнёрах для игры. Пользователь может просматривать профили и принимать решение о дальнейшем взаимодействии.

При заинтересованности пользователь имеет возможность поставить лайк выбранному профилю. Лайк является односторонним действием, то есть пользователь может выразить интерес независимо от решения другого человека.

После получения встречного лайка система автоматически фиксирует взаимное совпадение. На данном этапе создаётся match, а пользователям становится доступен встроенный чат.

Наличие механизма взаимного совпадения обеспечивает дополнительную безопасность и позволяет ограничить общение только между пользователями, проявившими взаимный интерес. Это снижает вероятность спама и повышает комфорт использования приложения.

После открытия чата пользователи могут обмениваться сообщениями, обсуждать совместную игру, договариваться о времени подключения и координировать дальнейшие действия.

Таким образом, алгоритм работы TeamUp можно представить в следующем виде:

**регистрация → авторизация → заполнение профиля → просмотр анкет → лайк → взаимный лайк → открытие чата → совместная игра**

Подобная структура обеспечивает логичную последовательность действий и делает взаимодействие пользователя с приложением максимально простым и понятным.

## 4.1 Использование встроенного чата

После формирования взаимного совпадения пользователям становится доступен встроенный чат. Основной задачей данной функции является обеспечение удобного общения между игроками без необходимости перехода в сторонние приложения или мессенджеры.

Система сообщений позволяет пользователям договариваться о совместной игре, обсуждать игровые стратегии, время подключения и другие организационные вопросы. Чат функционирует только между пользователями, которые поставили друг другу взаимные лайки. Такой подход позволяет минимизировать вероятность получения нежелательных сообщений и делает взаимодействие внутри приложения более безопасным и комфортным.

При отправке сообщения мобильное приложение формирует HTTP-запрос и передаёт его серверной части. Сервер обрабатывает полученную информацию, сохраняет сообщение в базе данных и отправляет его получателю. Каждое сообщение содержит текст, информацию об отправителе и времени отправки.

Интерфейс чата был разработан с учётом удобства мобильного использования. Пользователь может быстро просматривать переписку, отправлять новые сообщения и получать ответы в режиме реального времени.

В случае отсутствия подключения к сети Интернет система уведомляет пользователя о невозможности отправки сообщения. После восстановления соединения работа приложения продолжается в штатном режиме.

Таким образом, встроенный чат является важной частью функциональности приложения TeamUp и обеспечивает полноценное взаимодействие между пользователями после взаимного совпадения.

## 4.2 Возможные ошибки при работе приложения

В процессе использования приложения могут возникать различные ошибки, связанные с подключением к серверу или некорректным вводом данных.

Наиболее распространённой проблемой является отсутствие соединения с Интернетом. В данном случае приложение не может получить доступ к серверной части и выполнить необходимые запросы. Пользователь получает уведомление о невозможности подключения.

Другой возможной ошибкой является ввод неверного логина или пароля при авторизации. В подобной ситуации сервер выполняет проверку введённых данных и возвращает сообщение об ошибке.

Также система выполняет проверку заполнения обязательных полей при регистрации и редактировании профиля. Если пользователь оставляет обязательные поля пустыми, приложение выводит соответствующее уведомление.

Наличие системы обработки ошибок позволяет повысить стабильность работы программного продукта и улучшить пользовательский опыт.

# 5. РУКОВОДСТВО ПРОГРАММИСТА

## 5.1 Структура проекта

Программный продукт TeamUp состоит из двух основных частей: серверной и клиентской. Каждая из частей выполняет отдельные функции и взаимодействует посредством REST API.

Серверная часть реализована на Django и включает следующие компоненты:

— модели базы данных;

— маршруты API;

— сериализаторы;

— обработчики запросов;

— систему авторизации;

— систему лайков и совпадений;

— модуль сообщений.

Основная логика приложения сосредоточена именно на серверной стороне. Здесь выполняется обработка пользовательских запросов, взаимодействие с базой данных и управление всеми внутренними процессами системы.

Клиентская часть представляет собой мобильное приложение, разработанное на React Native. В состав клиентской части входят:

— экраны авторизации и регистрации;

— профиль пользователя;

— экран поиска игроков;

— экран совпадений;

— встроенный чат;

— система навигации.

Для взаимодействия с сервером используется библиотека Axios, выполняющая отправку HTTP-запросов.

Проект организован по модульному принципу, что значительно упрощает сопровождение и дальнейшую модернизацию приложения.

## 5.2 Запуск серверной части

Для запуска серверной части необходимо установить Python и все зависимости проекта. Установка зависимостей осуществляется через менеджер пакетов pip.

После установки библиотек необходимо выполнить миграции базы данных. Миграции создают необходимые таблицы SQLite и подготавливают систему к работе.

Запуск сервера осуществляется стандартной командой Django. После запуска сервер начинает принимать HTTP-запросы от мобильного приложения.

Для корректной работы мобильного приложения необходимо настроить CORS. В проекте использовалась библиотека corsheaders, обеспечивающая обработку междоменных запросов.

Дополнительно серверная часть может быть запущена в production-режиме с использованием Gunicorn.

## 5.3 Запуск клиентской части

Клиентская часть приложения разрабатывалась с использованием React Native и платформы Expo. Для запуска проекта необходимо установить Node.js и менеджер пакетов npm.

После установки зависимостей выполняется запуск Expo-сервера, обеспечивающего сборку и тестирование мобильного приложения. Expo значительно упрощает процесс разработки, поскольку позволяет запускать приложение без необходимости ручной настройки Android Studio на ранних этапах разработки.

Мобильное приложение взаимодействует с серверной частью посредством REST API. Для отправки HTTP-запросов используется библиотека Axios. Все запросы к серверу организованы централизованно, что упрощает поддержку проекта и позволяет изменять конфигурацию подключения без необходимости переработки всех компонентов приложения.

Навигация внутри приложения реализована с использованием стандартных механизмов React Native. Пользователь может переходить между экранами регистрации, авторизации, поиска игроков, совпадений и чатов.

Каждый экран приложения представляет собой отдельный компонент. Такой подход обеспечивает разделение логики интерфейса и делает структуру проекта более понятной.

После завершения разработки приложение было собрано в APK-файл под названием **teamup.apk**, предназначенный для установки на Android-устройства.

## 5.4 Развёртывание приложения

Развёртывание серверной части осуществлялось на VPS-хостинге Beget. Использование виртуального частного сервера обеспечивает постоянную доступность приложения и возможность обработки пользовательских запросов в круглосуточном режиме.

В качестве сервера приложений использовался Gunicorn. Данный инструмент предназначен для запуска Python-приложений и обеспечивает взаимодействие между Django и веб-сервером Nginx.

Nginx выполняет функции обратного прокси-сервера. Он принимает входящие HTTP-запросы и перенаправляет их Gunicorn, который, в свою очередь, взаимодействует с Django-приложением.

Подобная архитектура широко используется при развёртывании современных веб-приложений, поскольку обеспечивает высокую производительность, стабильность и безопасность работы системы.
